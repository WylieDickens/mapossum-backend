Welcome to the Mapossum Services API below is API doc

List of Methods:

/verify

requires:

email
password

returns if successful:

last
first
userid
affiliation
email
location (as text)

returns if unsuccessful:

email: will be empty string
userid: -1
message: "user not found"

/adduser

requires:

email
password
affiliation
first
last
location (wkt, LAT LON 4326 WGS84) default is: Point(39.828127 -98.579404) (center of US)

returns if successful:

last
first
userid
affiliation
email
location (as text)
success (True)

returns if unsuccessful:

success (False)

/updateuser

requires: (All values except userid are NEW)

email
password
affiliation
first
last
location
userid

returns if successful:

last
first
userid
affiliation
email
location (as text)
success (True)

returns if unsuccessful:

success (False)

/addquestion

requires:

userid
question

optional:

type: string, valid values are category,multiple,ordinal,numeric (multiple is default, category is same as multiple but allows "other")
hashtag: default is an empty string

returns if successful:

userid
qid
question
hashtag
type
success

returns if unsuccessful:

success (False)

/addanswer

requires:

qid
answer

optional:

color: string, can be rgb(XX,XX,XX) or #XXXXXX format

returns if successful:

qid
answer
answerid
color

returns if unsuccessful:

success (False)

/addresponse

requires:

qid
answerid (Not technically required but highly recommended (especially with multiple choice questions)
response (Only required if answerid is not given)
location as WKT LAT LONG WGS84

returns if successful:

qid
answerid
response
location as WKT LAT LONG

returns if unsuccessful:

success (False)

/getanswers

requires:

qid

returns if successful:

data (a json table of the answers, this table now also includes the number of time each question has been answered.)

returns if unsuccessful:

success (False)

/getquestions

requires:

count: a number representing the limit of questions default is 10
minutes: a number representing the number of minutes from the moment of request that you would like to get questions from

optional

hasanswers: default false, if true will only return those questions that have answers. Will also return the count (number of answers). Any value besides true is considered false.

returns if successful:

data (a json table of the questions with fields userid, question, type, hashtag, qid, and created)

returns if unsuccessful:

success (False)
